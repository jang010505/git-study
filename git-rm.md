## <code>git rm</code>란?
프로젝트를 진행하다 보면 파일을 삭제하고 싶은 경우가 있습니다. git에서의 파일 삭제는 일반적인 파일 삭제와 살짝 다릅니다. 중요한 점은 파일 삭제를 한 후 commit을 해주어야 합니다. 커밋은 한 시점의 스냅샷이므로, 파일을 삭제한 후 변경된 스냅샷을 저장해야 합니다.
### 사용 예시
<code>git rm "filename"</code> : 로컬 디렉토리와 git 저장소 모두에서 파일을 삭제할 수 있습니다. 예시는 다음과 같습니다:</br>
<code>git rm --cached "filename"</code> : 로컬 디렉토리에서는 파일을 유지하고 저장소에서만 삭제할 수 있습니다.</br>
